Portfolio

Project 1: Multi-Car Racing Simulator

The Multi-Car Racing Simulator is a dynamic program that animates several cars racing on a track. Initially developed as a single-car animation, this project was expanded to feature multiple cars competing simultaneously, as covered in Unit 5. Each car's progress is monitored, and upon crossing the finish line, their individual score is incremented, reflecting the number of races won. This project involved complex logic to ensure accurate detection of winners and the ability to reset and restart the race with a simple keypress.

Features:

  • Conversion of a single-car program to a multi-car array for simultaneous racing.
  • Implementation of a scoring system within the Car class to track race wins.
  • Methods to accurately detect winners and handle ties with precise calculations.
  • Enhanced user interaction with race reset functionality.

This application showcases object-oriented programming principles, array handling, and event-driven interaction within a graphical environment.

Project 2: UML Class Diagrams Interpretation

This project focuses on the interpretation and implementation of UML Class Diagrams into Java code. A UML diagram was provided, illustrating the relationships and structures of various classes, which I then translated into a functioning Java program. The UML diagrams were used to design the classes, variables, constructors, and methods according to the specified visibility and types.

Key Aspects:

  • Analysis of UML Class Diagrams and conversion into Java code.
  • Understanding of UML notation, including class variables, constructors, and methods.
  • Adherence to visibility rules indicated by UML's '+' (public) and '-' (private) symbols.
  • Implementation of the relationships between classes, such as composition, aggregation, and inheritance.

This project helped solidify my understanding of object-oriented design principles and their practical application in software development.
